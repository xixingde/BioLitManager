# 变更：实现前端首页功能

## 原因

当前系统首页只是一个简单的占位卡片，缺少必要的信息展示和快捷入口，影响用户体验。作为文献管理系统，首页应提供数据概览、快捷操作和待办事项，帮助用户快速了解系统状态并高效开展工作。

## 变更内容

- **新增首页统计卡片区域**：展示论文总数、待审核数量、已通过数量、我的草稿等关键指标
- **新增快捷操作入口**：提供录入论文、批量导入、我的论文等快捷入口
- **新增最近动态区域**：展示最近录入的论文和待审核任务
- **优化布局设计**：采用现代化的仪表盘式布局，使用 Ant Design 组件实现响应式设计

## 影响

- **受影响的规范**：前端页面规范
- **受影响的代码**：
  - `src/pages/home/HomePage.tsx`: 实现完整的首页功能
  - `src/services/statisticsService.ts`: 新增统计服务（使用前端聚合方式）
  - `src/types/statistics.ts`: 新增统计相关类型定义

## 实现方案

### 技术方案
1. 使用 Ant Design 组件构建首页（Card, Row, Col, Statistic, Table, List, Avatar 等）
2. 创建统计服务，使用现有 API 进行前端数据聚合
3. 根据用户角色动态展示审核入口
4. 实现响应式布局，适配不同屏幕尺寸

### 数据获取策略
- 使用现有的 `getPapers` 接口获取论文列表，在前端进行状态统计
- 使用 `getPendingBusinessReviews` 和 `getPendingPoliticalReviews` 获取待审核数据
- 使用 `getMyPapers` 获取当前用户的论文
