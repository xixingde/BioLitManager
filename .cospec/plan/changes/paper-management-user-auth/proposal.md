# 变更：基础架构搭建与用户权限系统

## 原因

实现论文管理系统的核心基础架构，建立前后端分离架构和基于 RBAC 的用户认证与权限管理，确保系统安全性和数据隔离。认证和权限控制是系统安全的基础，没有认证就无法保护论文数据的安全，无法实现不同角色之间的权限隔离。

## 变更内容

- **BREAKING**: 创建全新的前后端分离架构项目，实现基础目录结构和技术栈集成
- 实现用户认证功能：登录、登出、会话管理（JWT Token，有效期2小时）
- 实现密码复杂度校验（至少8位，含大小写字母、数字、特殊字符）
- 实现登录失败锁定机制（连续5次失败锁定1小时）
- 实现 RBAC 权限模型，支持7类角色（超级管理员、管理员、部门主管、课题组长、业务审核员、政工审核员、普通用户）
- 实现用户账户管理功能（禁用/启用账户）
- 实现权限校验中间件和路由保护
- 实现内存缓存机制（用户会话、热点数据）
- 实现 SQLite 数据库初始化和用户表创建
- 实现统一的错误码和响应格式

## 影响

- **受影响的规范**：用户认证与权限管理（spec.md 用户故事1，FR-001 至 FR-007）
- **受影响的代码**：
    - `src/frontend/`: 前端 React 应用架构（全部新建）
        - `src/frontend/src/`: 前端源码目录
        - `src/frontend/package.json`: 前端依赖配置
        - `src/frontend/vite.config.ts`: Vite 构建配置
        - `src/frontend/tsconfig.json`: TypeScript 配置
    - `src/backend/`: 后端 Go 应用架构（全部新建）
        - `src/backend/cmd/server/main.go`: 后端入口
        - `src/backend/go.mod`: Go 依赖配置
        - `src/backend/internal/`: 后端源码目录
    - `data/`: SQLite 数据库文件存储（新建）
